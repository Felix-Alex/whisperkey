## 2. 产品概述

### 2.1 产品名称建议

| 候选 | 中文 | 英文 | 推荐 | 理由 |
|---|---|---|---|---|
| 1 | **语灵听写** | **WhisperKey** | ⭐ 推荐 | "语灵听写"暗示"按键即说话、轻量私密"；英文 WhisperKey 双关 Whisper（语音模型+轻声）+ Key（快捷键），全球辨识度高，域名/商标可获得性较好 |
| 2 | 言出即文 | VoxType | | 名字更书面化，但稍长不易传播 |
| 3 | 语点 | VoiceDot | | 简短，但与现有产品撞名风险偏高 |

**最终决定**：产品名 = **语灵听写（WhisperKey）**，下文统一使用此名称。

### 2.2 产品定位

一句话定位：**"按一下快捷键，开口说话，文字直达光标处"——Windows 11 上最简单的全局 AI 语音输入工具。**

### 2.3 核心价值

1. **零学习成本**：用户全程只需记住 1 个快捷键。
2. **全局通用**：在所有可输入文字的位置（聊天、文档、邮件、表单、IDE、搜索框）都可使用。
3. **三档智能输出**：原话还原 / 智能润色 / Markdown 提示词，覆盖从随手输入到 AI 提示词工程的全场景。
4. **首启即用**：内置官方免费密钥，下载即可用，无需注册、无需配置。

### 2.4 用户画像

| 画像 | 描述 | 核心诉求 |
|---|---|---|
| 办公白领（主） | 25-45岁，每日处理大量微信/邮件/文档/表单 | 输入更快、减少打字疲劳 |
| 自媒体/创作者 | 写公众号/短视频脚本/播客文稿 | 把口述快速变文字稿 |
| AI 重度用户 | 频繁使用 ChatGPT/Claude/Cursor 写 Prompt | 把口述转高质量提示词 |
| 视障/特殊人群 | 因身体原因依赖语音 | 高识别准确率 + 无障碍 |

## 5. 功能需求清单

> 优先级说明：**P0** = MVP 必须；**P1** = V1.0 必须；**P2** = V1.1+ 增强。

### 5.1 唤醒与录音（P0）

| 字段 | 内容 |
|---|---|
| **ID** | F-001 |
| **名称** | 全局快捷键启停录音 |
| **描述** | 用户在任意 Windows 应用中按下快捷键即开始录音，再次按下停止录音并触发后处理 |
| **输入** | 默认快捷键 `Ctrl + Shift + Space`（用户可在设置中改为任意 1 个修饰键 + 主键的组合，必须包含至少 1 个 `Ctrl/Alt/Shift/Win` 修饰键） |
| **处理** | 1) 全局键盘钩子捕获组合键<br>2) 切换录音状态机：空闲→录音 / 录音→处理<br>3) 显示/隐藏屏幕底部中央的悬浮波形指示器 |
| **输出** | 录音 PCM 数据流（16kHz / 16bit / 单声道） |
| **异常** | 1) 快捷键被其他应用占用：注册失败时弹窗提示并要求改键<br>2) 麦克风未授权或被占用：托盘气泡 + 跳转设置引导<br>3) 录音超过 60 秒自动停止并继续后处理（防止误开） |
| **验收标准** | a) 在记事本/微信/Word/Chrome/Cursor 五个场景下分别按快捷键，均能正常启停录音<br>b) 启停响应延迟 ≤ 200ms<br>c) 默认快捷键不与 Win11 系统快捷键冲突 |

### 5.2 录音状态指示器（P0）

| 字段 | 内容 |
|---|---|
| **ID** | F-002 |
| **名称** | 录音悬浮波形指示器 |
| **描述** | 录音期间在屏幕底部中央显示一个小型悬浮窗，宽 200px、高 40px，圆角胶囊型，半透明背景，实时显示音频波形条，并在最右侧显示当前模式徽标（"原话/优化/MD"），处理中时切换为旋转 spinner |
| **输入** | 实时音频电平值 / 当前模式 / 处理状态 |
| **处理** | 始终最顶层显示，跨所有桌面，不抢焦点（WS_EX_NOACTIVATE） |
| **输出** | 视觉反馈 |
| **异常** | 多显示器场景下显示在主屏幕底部中央 |
| **验收标准** | 1) 不影响下方应用焦点和输入<br>2) 在游戏全屏（边框型）下也可见<br>3) 60Hz 流畅刷新 |

### 5.3 三种 AI 输出模式（P0）

| 字段 | 内容 |
|---|---|
| **ID** | F-003 |
| **名称** | 三种处理模式切换 |
| **描述** | 用户可在托盘菜单或设置面板切换"原话 / 优化 / Markdown"三种输出模式 |
| **模式 A 原话** | ASR 转写后调用 LLM 仅做：纠正同音字 / 加标点 / 识别中英混合；**禁止增删改任何语义** |
| **模式 B 优化** | LLM 二次处理：去除"嗯/呃/啊/那个/这个/就是说"等口语填充词；合并重复表达；优化语序；**严禁新增原话外信息** |
| **模式 C Markdown** | LLM 将口述需求转为结构化 Markdown 格式提示词，包含"# 任务 / ## 背景 / ## 要求 / ## 输出格式"等节段 |
| **输入** | ASR 文本 / 当前模式 |
| **输出** | 处理后文本 |
| **异常** | LLM 调用失败时：模式 A 直接输出 ASR 原文；模式 B/C 失败弹窗询问"是否回退为原话输出" |
| **验收标准** | 1) 三种模式输出风格在测试集 50 条样本上人工判定通过率 ≥ 90%<br>2) 模式切换后 1s 内生效 |

### 5.4 文本注入到光标（P0）

| 字段 | 内容 |
|---|---|
| **ID** | F-004 |
| **名称** | 全局光标位置文本注入 |
| **描述** | 处理完成后将文本注入当前焦点输入框 |
| **处理** | 主方案：写入剪贴板 + 模拟 `Ctrl+V`；备份用户原剪贴板内容并在 200ms 后还原<br>回退方案：若粘贴失败检测，使用 `SendInput` 逐字符 Unicode 输入 |
| **异常** | 1) 焦点丢失（无可输入控件）：写入剪贴板 + 托盘气泡"已复制到剪贴板"<br>2) 目标程序禁用粘贴：自动回退至 SendInput |
| **验收标准** | 在记事本/Word/微信/浏览器搜索框/VS Code/Excel 单元格全部测试场景成功率 100% |

## 7. 核心交互流程

### 7.1 主流程（成功路径）

1. 用户在任意应用焦点位置按 `Ctrl+Shift+Space`
2. 屏幕底部中央显示悬浮波形指示器（红点 + 实时波形）
3. 用户开口说话
4. 用户再次按 `Ctrl+Shift+Space`（或语音停顿超 3s 自动停止）
5. 指示器从"录音"切换为"处理中"（spinner 动画）
6. 后台并发：保存当前剪贴板 → 调用 ASR → 调用 LLM（按当前模式）→ 写入剪贴板 → 模拟 Ctrl+V → 还原原剪贴板
7. 指示器消失，文本出现在光标位置
8. 历史记录入库

### 7.2 异常分支

| 场景 | 处理 |
|---|---|
| 麦克风权限未授权 | 弹出"系统设置 → 隐私 → 麦克风"指引 |
| 网络断开 | 指示器变红色 X + 托盘气泡"网络异常，请检查后重试" |
| ASR 超时（>10s） | 取消请求 + 提示"识别超时" |
| LLM 超时（>15s） | 模式 A 用 ASR 原文兜底；模式 B/C 询问"使用原话还是重试" |
| 焦点丢失 | 文本仅写剪贴板 + 托盘气泡"已复制到剪贴板（Ctrl+V 可粘贴）" |
| 快捷键冲突 | 启动时检测注册失败 → 引导用户改键 |
| 余额不足/Key 失效 | 弹窗显示具体错误 + 跳转配置 |

## 9. 非功能需求

| 类别 | 指标 |
|---|---|
| **启动性能** | 冷启动到托盘可见 ≤ 1.5s（SSD）；首次快捷键响应 ≤ 200ms |
| **录音延迟** | 按下快捷键到指示器显示 ≤ 100ms |
| **端到端时延** | 5 秒录音的"按停止键到文字落地"全流程 ≤ 3.5s（P50） |

| **无障碍** | 全键盘操作；屏幕阅读器友好（V2.0） |

## 10. 商业化与许可证机制

### 10.1 价格

| 商品 | 首发价 | 标准价 |
|---|---|---|
| 优化模式 | ¥35 | ¥39 |
| Markdown 模式 | ¥35 | ¥39 |
| 双模式套餐 | ¥45 | ¥59 |

### 10.2 激活码生成（服务端，仅描述逻辑）

1. 服务端使用 RSA-2048 私钥
2. 激活码由订单生成：`{orderId, productCodes, validUntil=lifetime, sig}` Base32 编码
3. 用户输入激活码 → 客户端联网到激活服务器换取**本地许可证文件**

### 10.3 本地许可证文件 license.dat（JSON）

```json
{
  "version": 1,
  "userId": "u_xxxxxxx",
  "products": ["polish", "markdown"],
  "deviceFingerprint": "sha256(...)",
  "issuedAt": 1736000000,
  "signature": "base64(RSA-PSS sign of payload by server private key)"
}
```

### 10.4 校验流程
- 启动 → 读取 license.dat → 用内置 RSA 公钥验签 → 验机器指纹 → 写入只读内存
- 离线可工作；每 30 天若联网则向激活服务器静默续签（防黑名单）

### 10.5 防破解策略

| 措施 | 描述 |
|---|---|
| 代码混淆 | Rust release + symbol strip |
| 关键判断多点 | 校验逻辑分散在 3 个不同模块 |
| 完整性校验 | 程序启动时校验自身 hash |
| 黑名单同步 | 每周可选拉取已泄露激活码黑名单 |
| 服务器端 | 单激活码最多 3 台设备同时活跃；超出锁定 |
| 定位 | 反破解到"提高破解成本"即可，不追求绝对安全 |

## 12. 风险与应对

| # | 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|---|
| 1 | 全局快捷键被杀软拦截 | 中 | 高 | 使用 RegisterHotKey（白名单友好）；首次安装引导用户加白名单 |
| 2 | 文本注入在某些应用失败（如部分 UWP/Electron） | 高 | 高 | 三级回退：剪贴板粘贴 → SendInput Unicode → 仅复制+气泡提示 |
| 3 | 官方免费密钥被滥用 | 高 | 中 | 中转服务器限频 + 设备指纹 + 总日配额硬限 |
| 4 | LLM 输出不符合三模式约束（特别是模式 B 增添信息） | 中 | 高 | 严格 System Prompt + 输出后正则二次校验"长度不超过原文 1.2 倍" |
| 5 | 第三方 API 厂商接口变更或下线 | 中 | 中 | 适配器模式 + 配置驱动；可热更新 endpoint 配置 |
| 6 | 中英混合识别准确率不达预期 | 中 | 中 | ASR 厂商 A/B 切换；提供"语种偏好"高级设置 |
| 7 | 激活码被破解导致付费模式被白嫖 | 高 | 中 | 接受 5-10% 破解率；专注提升合法用户体验+服务 |
| 8 | Win10 兼容性问题 | 低 | 中 | CI 上 Win10/Win11 双环境冒烟测试 |
